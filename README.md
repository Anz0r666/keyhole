# 🔑 Keyhole

**Платёжный + идентификационный слой с программируемыми правилами для автономных ИИ-агентов.**
Когда не человек, а его ИИ-агент нажимает «купить» — Keyhole отвечает на три вопроса: *кто это, что ему можно, и кто отвечает.*

> ⚠️ Это **ТЕСТОВЫЙ режим (sandbox)**. Никаких реальных денег, банков и чужих ключей. Всё живёт в памяти процесса. Идеально, чтобы показать суть.

---

## Что это демонстрирует — 4 «ворот» каждой операции

| Ворота | Что проверяется |
|--------|-----------------|
| 🪪 **Личность** | У каждого агента настоящая криптопара (Ed25519). Агент подписывает операцию, Keyhole проверяет подпись. Самозванец отсекается. |
| 🎛️ **Правила** | Дневной лимит, белый/чёрный список категорий, порог авто-подтверждения — зашиты на уровне платежа, а не «честного слова» модели. |
| 👤 **Подтверждение** | Крупная трата (выше порога) «зависает» и ждёт зелёного света человека. |
| 🤝 **Доверие агент↔агент** | Сделка между двумя агентами проходит проверку репутации контрагента. |

## Запуск

Нужен только **Node.js** (никаких `npm install` — ноль зависимостей).

### Вариант 1 — веб-дашборд
```bash
node server.js
```
Открой **http://localhost:4178** → нажми «▶ Запустить демо-сценарий».

Внутри дашборда есть **интерактивный конструктор**: создай своего агента с любыми правилами и проводи произвольные траты руками — увидишь, как Keyhole пропускает, блокирует или отправляет на подтверждение.

### Вариант 2 — демо прямо в терминале
```bash
node demo.js
```

### Вариант 3 — автономный агент через SDK
```bash
node server.js          # в одном окне
node example-agent.js   # в другом — агент сам закупается через Keyhole
```

### Pitch deck
Открой **http://localhost:4178/deck** — презентация из 12 слайдов (листается стрелками ← →).

## Демо-сценарий (что увидишь)

ShopperBot (твой агент, баланс $500, лимит $200/день) пытается тратить:

| Трата | Результат | Почему |
|-------|-----------|--------|
| $20 · cloud | ✅ ALLOW | в рамках правил |
| $300 · cloud | ⛔ DENY | превышен дневной лимит |
| $10 · gambling | ⛔ DENY | категория в чёрном списке |
| $75 · data | ⏳ → ✅ | выше порога → подтверждено человеком |
| $30 · saas → SupplierBot | ✅ ALLOW | сделка агент↔агент, доверие подтверждено |

## Структура

```
keyhole/
├── server.js         — HTTP API + раздача дашборда и презентации (чистый Node)
├── engine.js         — операционный движок (async): подпись → правила → доверие → рельсы
├── policy.js         — движок правил (сердце продукта)
├── validate.js       — валидация входных данных (первый рубеж обороны)
├── security.js       — auth · rate limit · лимит тела · security-заголовки
├── rails.js          — адаптер платёжных рельсов (sandbox / Stripe / стейблкоины)
├── store.js          — хранилище + персистентность + API-ключи + криптоличности
├── keyhole-sdk.js    — SDK для разработчиков агентов (подключение в 3 строки)
├── example-agent.js  — пример автономного агента, который сам платит через SDK
├── demo.js           — демо в терминале
├── test/
│   ├── keyhole.test.js  — автотесты движка (node --test)
│   └── security.test.js — тесты безопасности (негативные кейсы)
├── SECURITY.md       — полный аудит безопасности
├── data/             — сохранённое состояние песочницы (в .gitignore)
├── public/
│   ├── index.html    — живой дашборд + конструктор агентов (Twilight UI)
│   └── deck.html     — pitch deck из 12 слайдов
└── README.md
```

## Безопасность

Полный аудит — в [SECURITY.md](SECURITY.md). Реализовано:

- 🔑 **API-ключи** — мутирующие эндпоинты требуют `Authorization: Bearer <key>`. Ключ печатается при старте и хранится в `data/state.json`. Дашборд получает ключ инъекцией.
- ✅ **Валидация входных данных** — отрицательные/`NaN`-суммы, инъекции в категории, мусорные строки отбиваются (закрыт критичный баг порчи баланса).
- 🚦 **Rate limiting** — лимит запросов на IP → `429`.
- 📦 **Лимит тела запроса** (256 КБ) → `413`.
- 🛡️ **Security-заголовки** — CSP, `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`.
- 💪 **Устойчивость** — глобальный перехват ошибок, `/health`; сервер не падает от плохого ввода.

SDK с ключом:
```js
const kh = new Keyhole('http://localhost:4178', process.env.KEYHOLE_KEY);
```

## Реальные платёжные рельсы (следующий этап)

Движение денег вынесено в адаптер `rails.js`. Сейчас работает `SandboxRails` (без реальных денег).
Чтобы подключить реальные рельсы — не трогая бизнес-логику:

```bash
STRIPE_KEY=sk_...   node server.js   # Stripe Issuing (каркас готов, нужен ключ)
RAILS=stablecoin    node server.js   # x402 / USDC (запланировано)
```

## API (если хочешь дёргать руками)

| Метод | Путь | Назначение |
|-------|------|-----------|
| `GET` | `/api/state` | текущее состояние (агенты, кошельки, леджер) |
| `POST` | `/api/scenario` | прогнать готовый демо-сценарий |
| `POST` | `/api/agents` | создать агента `{name, owner}` |
| `POST` | `/api/wallets` | задать кошелёк и правила `{agentId, balance, rules}` |
| `POST` | `/api/pay` | провести трату `{agentId, amount, category, merchant, counterpartyAgentId?}` |
| `POST` | `/api/approve` | подтвердить зависшую трату `{approvalId}` |
| `POST` | `/api/reset` | сброс песочницы |
