# 🛡️ Keyhole · Аудит безопасности

Полный реестр уязвимостей, найденных при аудите кода, с уровнями критичности и статусом исправления.
Дата аудита: 2026-06-23 · Режим: тестовый (sandbox).

Легенда статуса: ✅ исправлено · 🟡 принято как ограничение тест-режима (задокументировано)

## 🔴 CRITICAL

### C-1 · Порча баланса отрицательной/`NaN` суммой ✅
**Было:** сумма не валидировалась. `processPay({amount: -1000})` проходил все правила
(`spentToday + (-1000) > limit` ложно) и выполнял `balance -= -1000` → **баланс рос из ничего**.
`amount: "abc"` → `Number("abc") = NaN`, `NaN > limit` ложно → проходил → `balance -= NaN` → **порча кошелька**.
**Фикс:** `validate.js` — сумма обязана быть конечным числом `> 0` и `<= 1e9`. Плюс защита в движке (defense-in-depth).

## 🟠 HIGH

### H-1 · Полное отсутствие аутентификации ✅
**Было:** любой, кто видит порт, мог создавать агентов, менять кошельки, проводить траты, делать reset.
**Фикс:** API-ключи (Bearer). Мутирующие эндпоинты требуют ключ. Дашборд получает ключ инъекцией в HTML.

### H-2 · Нет защиты от DoS (флуд запросами) ✅
**Было:** бесконечные запросы / создание миллионов агентов клали бы сервис и память.
**Фикс:** rate limiter (лимит запросов на IP в окне) → `429 Too Many Requests`.

### H-3 · Нет лимита размера тела запроса ✅
**Было:** запрос на гигабайты JSON копился в память (`data += c`) → исчерпание памяти.
**Фикс:** лимит тела (по умолчанию 256 КБ) → `413 Payload Too Large`.

### H-4 · Необработанное исключение роняет/вешает запрос ✅
**Было:** ошибка в обработчике → необработанный промис, запрос висит, риск падения.
**Фикс:** глобальный `try/catch` в роутере + `uncaughtException`/`unhandledRejection` без выхода. Сервер не падает.

## 🟡 MEDIUM

### M-1 · Нет валидации строк/категорий (мусор, контрол-символы) ✅
**Было:** имя/категория любой длины и содержимого — риск замусоривания, ломки логики, XSS в дашборде.
**Фикс:** `validate.js` — лимиты длины, очистка контрол-символов, категория по regex `^[a-z0-9_-]{1,40}$`.

### M-2 · Self-dealing: агент — контрагент сам себе ✅
**Было:** `counterpartyAgentId === agentId` накручивал репутацию (+2 за операцию).
**Фикс:** запрет сделки с самим собой в движке.

### M-3 · Нет security-заголовков ✅
**Было:** ни CSP, ни `X-Content-Type-Options`, ни анти-clickjacking.
**Фикс:** `Content-Security-Policy`, `X-Content-Type-Options: nosniff`, `X-Frame-Options`, `Referrer-Policy`.

### M-4 · `reset` / `scenario` стирают всё состояние без защиты ✅
**Было:** открытый `POST /api/reset` — любой обнулял данные.
**Фикс:** под аутентификацией (см. H-1).

## 🔵 LOW / INFO

### L-1 · Приватные ключи агентов хранятся на диске в открытом виде 🟡
**Статус:** ограничение тест-режима. `data/` в `.gitignore`, ключи не отдаются через API (`snapshot` их вырезает).
**В проде:** ключи держит сам агент (подпись на стороне клиента) либо HSM/KMS; сервер хранит только публичные.

### L-2 · «Ворота личности» сейчас театральны 🟡
**Статус:** в тест-режиме сервер и подписывает, и проверяет своим же ключом — проверка всегда проходит.
**В проде:** агент подписывает операцию приватным ключом у себя, Keyhole проверяет только публичным. Архитектура (`verifyCharge`) уже готова к этому.

### L-3 · Нет HTTPS 🟡
**Статус:** локальный тест-режим. **В проде:** TLS-терминация на реверс-прокси (nginx/Caddy) или платформе.

### L-4 · CORS не ограничен явно ✅
**Фикс:** ответы без разрешающих CORS-заголовков (только same-origin дашборд). Кросс-сайт запросы из браузера блокируются.
